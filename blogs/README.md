## Writing Guidelines

### Target Audience
- **Primary**: Developers, engineers, and scientists at frontier AI labs, AI startups, and enterprise AI teams
- **Secondary**: Product managers and QA engineers interested in AI agent testing
- **Tone**: Technical but accessible, personable, sharing genuine learnings

### Content Principles
1. **Build in Public**: Share both successes and challenges
2. **Technical Depth**: Provide real code examples and architectural insights
3. **Practical Value**: Focus on use cases and real-world applications
4. **Behind the Scenes**: Share unique learnings about vibe coding and Claude Code
5. **Community Focus**: Invite feedback, questions, and contributions

### Post Structure
1. TL;DR or quick summary
2. Context and problem statement
3. What we built (technical details)
4. Behind-the-scenes learnings
5. Next steps and roadmap
6. Clear call to action

### Distribution Checklist
For each post:
- [ ] LinkedIn post
- [ ] Twitter/X thread
- [ ] Reddit (r/MachineLearning, r/LocalLLaMA)
- [ ] HackerNews (when appropriate)
- [ ] Dev.to or Medium
- [ ] Update README.md with post link
